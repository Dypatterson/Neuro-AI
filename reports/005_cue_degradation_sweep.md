## Finding

This sweep varies temporal cue quality while keeping the content query as the
exact `stair` vector. That makes the result easier than the next real test:
content still anchors the correct item strongly.

Result shape:

- Exact, partial, and noisy temporal cues all recover the correct temporal
  neighborhood.
- Low temporal beta (`0.5-2`) leaves the state ambiguous but correct.
- Temporal beta `4+` commits cleanly for correct/noisy cues.
- A wrong temporal cue (`ladder + paint`) flips the anchor to `ladder` at
  temporal beta `4+`.
- Pure noise does not hurt because exact content still anchors `stair`.

Architectural implication: temporal cue degradation alone is not enough. The
next real stress test must degrade content and temporal cues together. That is
closer to live use, where a query rarely arrives as an exact stored vector.
