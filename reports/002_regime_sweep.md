# MVP 0.2 Regime Sweep

Generated by:

```bash
PYTHONPATH=src:. python3 experiments/regime_sweep.py
```

The sweep maps beta, temporal-window size, and semantic-family tightness across
two probes:

- `temporal_shuffle`: ordered stream vs. shuffled order control
- `content_distractor`: temporal recall vs. content nearest-neighbor when
  `stair/ladder/ramp/step/escalator` are intentionally nearby in FHRR space

## Findings

The clean temporal-shuffle probe has a sharp transition: beta `1-2` is a blend
regime with near-max entropy and little temporal signal; beta `4+` recovers the
ordered temporal structure across all tested windows.

The distractor probe exposes the important limitation. When content distractors
are almost identical (`family_noise = 0.10`), this MVP cannot recover the lived
association at any beta. The content basin is too entangled. As the family
spreads, robust temporal recall appears:

| Family noise | First robust beta, window 1 | First robust beta, window 2 | First robust beta, window 3 | First robust beta, window 4 |
|---:|---:|---:|---:|---:|
| 0.10 | none | none | none | none |
| 0.18 | 100 | 100 | 200 | 200 |
| 0.35 | 32 | 32 | 32 | 32 |
| 0.60 | 4 | 16 | 16 | 16 |

Interpretation: beta can rescue temporal specificity only when the content
patterns remain separable enough for the Hopfield content channel to commit to
the correct anchor. If several anchors are too close, their temporal contexts
blend before the temporal channel gets a clean read. This is the first concrete
pressure for either better separability, an explicit anchor disambiguation step,
or a future energy term that jointly scores content and temporal consistency
rather than reading temporal context after content settling.
